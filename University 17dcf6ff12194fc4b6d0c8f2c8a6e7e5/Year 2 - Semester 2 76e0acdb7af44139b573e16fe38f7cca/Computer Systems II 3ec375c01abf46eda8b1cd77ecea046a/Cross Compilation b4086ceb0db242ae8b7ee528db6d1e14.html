Cross Compilation

‘Normal’ Compilation:

Cross Compilation:

  • Used when compilation on target is impossible or impractical:
    • First compiler on new hardware or new OS
    • Low capability targets → embedded systems
    • (i.e. when there isn’t enough RAM space or the embedded system will take too long to compile - 1 day)
  • Architecture of host and target may be different:
    • Memory architecture (Neumann vs. Harvard)
    • Word size (64 bit vs. 8 bi)
    • Order of bytes in word

    Build Process

  • Unlikely things to go wrong when going from compiler to assembler - however, people who really optimise stuff can tweak the compiler until it produces what they like
  • Linker will find out if there are components missing in your program (functions, variables)
  • Tough to document what the compilation process/make file is when using IDEs, therefore not recommended to use when programming with embedded systems
  • With embedded systems, have to assume that bits WILL flip in your system…
    • Optimise it too much: will not write the same value twice
    • Don’t optimise it enough: will take too long
  • Can document easily when writing directly from command line, not IDE. Makes things a lot clearer

Linker Output: .elf object file

Object File Types:

  • Relocateaable (.o)
  • Executable (a.out/exe)
  • Shared object (.so/dll)

elf → Executable and Linkable Format

Platform independent binary interface for object files

Contains layout information and sections

  • .text - reads instructions
  • .rodata - read only data
  • .data - contains all initialised global variables
  • .bss - contains any undeclared variables that get zeroed out

static vs. local vs. global variable:

global variables are in memory which is fixed allocated

local variables go on the stack (meaning they get removed)

static go into memory, but can’t be accessed by other functions that haven’t declared variable

C zeroes out global variables but not local variables because: C can do this in one big block because all global variables are declared at start together whereas local variables are scattered, within functions
