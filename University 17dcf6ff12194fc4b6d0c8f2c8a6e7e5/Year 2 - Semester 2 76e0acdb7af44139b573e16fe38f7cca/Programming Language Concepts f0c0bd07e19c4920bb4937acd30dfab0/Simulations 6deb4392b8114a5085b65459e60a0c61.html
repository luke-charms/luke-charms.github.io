Simulation introduction

Previously, we considered trace equivalence as a possible way of comparing concurrent behaviour

  • Branching points in behaviour are not captured by traces

Can we define a notion of equivalence that accounts for branching points?

This is the idea behind simulation :

A state y in a labelled transition system can simulate a state x if

  • Whenever x can do some action, becoming x’ in the process, y can reply with the same action, becoming y’ in the process
  • Moreover, now y’ can still do everything that x’ can do ... and so on

This latter point is how we address the branching - any observations possible of x' must still be possible in the matching y' state

This is closely related to the notion of refinement in formal systems such as Event B

Simulations (more formally)

  • Remember: A binary relation on a set X is simply a subset R ⊆ X x X
  • Suppose that L = (X, Σ, L) is a labelled transition system (LTS)
  • A binary relation R on states of L is called a simulation on L if R satisfies the following condition:
  • For example, the identity relation Id = { (x,x) | x in X } is always a simulation on L
  • We define a simulation between two labelled transition systems simply by considering the above definition on the disjoint union of the two LTSs

Similarity

Theorem: The union of two simulation relations R1, R2 on an LTS L is a simulation relation on L

Proof: To show this we take any (x,y) ∈ R1 ⋃ R2, then (x,y) is either in R1 or R2

Suppose, wlog that it is in R1. Then suppose xa→x’x \frac a \rightarrow x’x→a​x’﻿ for some a and some x' We know that R1 is a simulation so there exists a ya→y’y \frac a \rightarrow y’ y→a​y’﻿ such that (x',y') ∈ R1 Hence (x',y') ∈ R1 ⋃ R2

Theorem: For any given LTS L there is a largest simulation on L

Proof: The largest simulation on L is the union of all simulations on L

Coinduction

Similarity is an instance of something called a coinductively defined relation in mathematics

  • It is the “largest relation such that…”

This gives us an easy proof technique for showing that something is in the similarity relation:

To show that (x,y) ∈ ≤ (written x ≤ y) for some states x,y of an LTS, we simply need to show that (x,y) ∈ R for some relation R that is a simulation

  • This is because:
    • If (x,y) ∈ R and R is a simulation, then because ≤ is the largest simulation then R ⊆ ≤ and hence x ≤ y also

This is known as the coinduction principle and using it gives us a coinductive proof technique

Using simulations:

So then, using all this information how do we:

  • Q. Show that y simulates x?
    • A. Construct a simulation that contains the pair (x,y)

  • Q. Show that y does not simulate x?
    • A. Play the simulation game of course!

The Simulation Game

Rules of the Game:

Game starts at position (x,y) and you are playing against the demon

  1. The demon picks a move starting from x, they choose xa→x’x \frac a \rightarrow x’x→a​x’﻿
  1. You must start from y and choose a matching move to y’ say ya→y’y \frac a \rightarrow y’ y→a​y’﻿
  1. Game goes back to Step 1, changing the position of (x’,y’)

If at any point, a player cannot make a move, that player loses

If the game goes on forever, you win

(The demon always plays moves reachable from x and we always play moves reachable from y)

Strategies

A strategy for the demon in this game is a mapping from every pair of states (x,y) to a move xa→x’x \frac a \rightarrow x’x→a​x’﻿

Similarly, a strategy for the non-demon player in this game is a mapping from every (x',y) and label a to a move ya→y’y \frac a \rightarrow y’ y→a​y’﻿

A strategy for a player is called a winning strategy if no matter which moves the other player makes, as long as they follow the winning strategy they are guaranteed to win

  • Theorem : For any two states, x ≤ y if and only if the non-demon player has a winning strategy in the simulation game
  • Proof : In essence you use the winning strategy to continue matching moves until all nodes reachable from x,y have been visited

Simulation Equivalence

  • States x and y are said to be simulation equivalent if both x ≤ y and y≤x

Thus to check that two states are simulation equivalent, we typically need to construct two simulations - one in each direction

  • e.g. We know that x0 and y0 below are not simulation equivalent, i.e., we have shown:
    • y0 ≤ x0, by constructing a simulation
    • but not x0 ≤ y0, by playing the simulation game

Simulation and Trace Equivalence

Theorem: If x and y are simulation equivalent then they are also trace equivalent.

Proof: Essentially, any trace of actions {a1a2 ... aN} starting from a state x can be matched using the simulation relation to give the same trace of actions from y, and v.v

  • We have seen that the converse is not true: there are trace equivalent states (x0 and y0) that are not simulation equivalent
