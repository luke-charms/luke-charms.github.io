DHCP (Dynamic host configuration protocol)

How systems get a dynamic IP address

Telnet

Simple unencrypted terminal emulation protocol

Currently not used across the internet now but used occasionally on local nets for configuring old hardware

Has been replaced by ssh, scp

Simple mail Transfer Protocol

  • Reliable protocol used to send email
  • Typically TCP connection to a mail server, with these commands/replies:
    • MAIL,RCPT,DATA
  • SMTP Authentication is an extension
    • Original email had none – so mail relays were used for spam

HTTP (Hypertext Transfer Protocol)

It’s a text-based protocol

  • TCP connect to server ip (normally) on port 80
  • Send request, e.g., GET /something/page.html HTTP/1.1
  • Receive response message with data - HTTP/1.1 200 OK (followed by payload)

Main HTTP requests:

  • GET
  • HEAD - like GET but no data is sent back
  • POST - send data to the server, could create
  • PUT - meant to change the resource on the server
  • DELETE - removes a resource

Error codes:

  • 1xx -> information
  • 2xx -> Success
  • 3xx -> Redirection (e.g. 301 moved)
  • 4xx -> Client error (e.g. 403 forbidden, 404 not found)
  • 5xx -> server error (e.g. 500 internal error)

Versions

version 2:

  • First major revision to HTTP, published in 2015
  • Header compression – now binary using HPACK
  • Server push
  • Multiplexing multiple requests over a single TCP connection

version 3:

  • Does not use TCP connections
  • Runs instead over QUIC with UDP and is widely in use (which helps with previous TCP issues)

QUIC - Quick UDP Internet Connections

  • UDP protocol to make web/etc faster – than TCP
  • Really a transport protocol – but is in User-space so... it can easily be upgraded
  • Google pushed it forward so LOTs of traffic is now QUIC - and IETF is standardising it
  • Still establishes some “state”/sessions and encryption

CoAP - Constrained Application Protocol

Provides HTTP-like protocol for simpler devices

  • Minimal overhead, good for IoT
  • Binary GET/PUT/… (like HTTP)
  • Simple discovery mechanism (GET a list of resources)
  • Simple Subscribe method (Used in Ikea smart lighting for example)

Real time audio/video streaming

Video frames can be “packetised” so that losses do not corrupt the whole stream

TCP and UDP can be used for transport (Clearly, however, UDP is a better choice)

HTTP can be used to stream media files

Real Time Streaming Protocol (RTSP)

Has a URL scheme with arguments (Used by Youtube to Flash players)

Real-time transport protocol (RTP) delivers the media (typically UDP)

SMB - File sharing protocols

Server Message Block

  • From Microsoft world
  • TCP port 445 or UDP ports
  • Provides authentication, file locking, etc…

NFS

Used mainly in Unix/Linux

Common between servers

Has to cope with very large files and file numbers
