Introduction

Security Mechanisms

Firewall

  • A network device that sits at the edge of network
  • Inspects each packet passing through it and matches them to a set of rules:
    • Packets that meet the requirements are passed through
    • Packets that don’t meet the rules are dropped or rejected
  • In a SOHO scenario: usually an embedded platform with other functions
  • In an enterprise scenario: might be combined with a router or implemented as a standalone device (bridging/transparent firewall)

Stateful Firewalls

  • Stateless firewalls are limited:
    • A rule to allow web traffic to any web server would allow an attacker to do anything from port 80 inbound
  • Modern firewalls are stateful
    • Inspect TCP/IP header fields and keep track of connections
    • Have rules for related and established
    • e.g. allow outbound traffic to port 80 but block inbound traffic from port 80 unless it is related to existing connection
  • A common rule set for (home) firewalls is:
    • Deny all inbound unless related / established
    • Allow all outbound.

Application Layer Firewall

  • Even more advanced firewalls can act at the application layer
    • May be known as “Layer 7” firewalls or Application Layer Gateways
  • Inspect the contents of the packet to make sure that that packet that looks like HTTP on port 80 is actually HTTP
  • Doesn't necessarily work with encrypted traffic:
    • e.g. SSH might "look" like HTTPs

Network Intrusion Detection

  • Sometimes people get into your network or legitimate users do malicious things
  • "How can you mitigate this?” Try to detect it.
  • Enter (N)IDS: a legitimate use of network sniffing
    • Monitors a network looking for malicious activity or policy violations
    • May be signature based or use anomaly-based detection
  • NIDS may be monitor and alert only, or it may be able to take action to prevent the intrusion.

802.1x →

  • Uses the Extensible Authentication Protocol (EAP) to authenticate clients.
  • Blocks the switch port/ wireless client until they authenticate successfully.
  • Involves 3 parties:
    • A supplicant (client)
    • An Authenticator (switch or access point)
    • An Authentication Server (e.g. LDAP using RADIUS)

IPSEC

  • NAC doesn’t stop unauthorised parties sniffing traffic if they tap a condition
    • For a 10/100, Hubs can be used as a tap
    • For Gigabit, a network tap (or port mirror) is required
    • Fibre is not immune
    • ARP cache poisoning can be used to get traffic as well
  • If traffic is sent “in the clear” over wire, then it can be sniffed
  • Solution: IP security - encryption at the Network layer
  • Authenticates and encrypts packets

Virtual Private Network (VPN)

  • A secure way to connect remote sites or users over an untrusted network, such as the internet
  • Two different kinds:
    • Site-to-Site: Securely connects two networks
    • Remote-access: individual clients connect to a VPN server
  • Creates an encrypted tunnel between the two endpoints
  • Used to provide secure remote access for employees or to connect branch offices
  • VPNs can use various protocols, such as:
    • Point-to-Point Tunneling Protocol (PPTP)
    • Layer Two Tunneling Protocol (L2TP)
    • Secure Socket Tunneling Protocol (SSTP)
    • Internet Protocol Security (IPsec)

Simple networks behave like hubs as every client receives all traffic

DNS Exploitation

Welcome to this tutorial on how to do a DNS amplification attack:

  1. Spoof source IP address to send lots of (small) DNS requests
  1. DNS server replies to spoofed address with lots of (large) DNS responses
  1. Destination is inundated with traffic and falls over
  1. ???
  1. PROFIT 🤑

Trusting DNS responses?

DNS is not signed or encrypted!

Can be taken further to DNS cache poisoning

  • Attacker supplies a bogus response to a caching resolver
  • All users of the resolver now use the bogus entry

OR, just a malicious network operator (include dodgy entries in their DNS server)

DNS Privacy

Your connection to a server may be encrypted i.e., what you are doing is obscured

Nevertheless, DNS is not encypted

  • Anyone sniffing the connection can see your DNS requests
  • therefore, where you are doing it is not obscured!

DNS Security Extensions

DNSSEC addresses DNS insecurity

  • Allows responses to be signed
  • BUT responses are larger and there exists complexity managing signatures

Other Improvements:

  • Mitigating DNS amplification - block resolver lookups from a non-local IP address and then restrict lookups to those from your organisation or customers only!
    • Ignore certain request types (<root> and ANY)

DDoS

Goal of DDoS attacks is to take a target system DOWN 👇🏼

Can be done by consuming:

  • Network bandwidth
  • CPU cycles
  • Server state capacity
  • Other system resources

Can also be used as a distraction for other cyber attacks:

  • Deploying malware
  • Stealing data

What is it?

There are effective forms of an attack from a single source system

A distributed attack may be more effective though:

  • Uses more systems to deprive the victim of the targeted resource (usually bandwidth)
  • Is harder to step as there are multiple sources (compromised computers/routers/people calling to action) and many of them may be expected sources in normal conditions

Example: TCP SYN Flood

Exploits the TCP protocol

  • Attacks the connection handshake
  • Attacker sends a TCP SYN packet
  • Server replies with SYN ACK
  • Attacker never sends final ACK so the victim is left waiting

Consumes state on server, especially many half-open connections which means that eventually cannot respond to legitimate requests

Low Orbit Ion Cannon (LOIC) and HOIC

A fairly crude UDP/TCP attack tool

Can be partially mitigated by blocking UDP and ICMP

(look more online in case)

SlowLORIS

Uses a single machine or even a mobile phone!

Opens lots of connections

Exact implementation depends on web server

Can be mitigated with (reverse) proxies, limiting the number of connections per-IP or increasing capacity

IPv6 RA Flood

Sends a large number of IPv6 RA multicasts

Consumes CPU on victim machines

“easy” to exploit

DDoS Mitigation

Anycast/content delivery networks: Spreads the load across lots of networks/systems

Responsive network team which can block attacks as they occur and redistribute services across the different connections

Make sure your systems are patched and configured securely
