Data Replication

Why replicating data?

  • Data objects replicated on more servers
  • Fault tolerance
    • Data still available if some server crashes
  • Load balancing
    • More servers can serve data requests
    • Servers can be placed closer to end-users

CAP Theorem

Commonly desired properties for a replicated data store:

  • Consistency
    • All replicas of a same data object has always the same state
  • Availability
    • Requests are served as long as at least one server is available
  • Partition Tolerance
    • The data stores keeps working even if servers are partitioned
Theorem states that: Impossible to achieve all three only two can be achieved at same time

AP Systems

  • In that case, the data store gives up atomic consistency to ensure availability and partition tolerance → AP System
  • What type of consistency can AP Systems guarantee?

    → Consider partially synchronous networks

    • An asynchronous network that may delay messages among correct nodes arbitrarily, but eventually behaves synchronously and delivers all messages within a fixed (but unknown) time bound
    • During “synchronous periods”, servers can reconcile data objects replicas to make them consistent
    • During “asynchronous periods”, read operations can return non-consistent versions
    • Eventual consistency

tldr:

  • Optimistic or Lazy replication performs synchronisation in the background
  • Possible for conflicts to arise
  • Resolving conflicts comes at a cost
  • Scales to large numbers of replicas due to lower burden of synchronising

CP Systems

  • There can be use cases where relying on inconsistent data might not be acceptable
  • In that case, the data store gives up availability to ensure consistency and partition tolerance → CP System

tldr: 1) Need to synchronise replicas to achieve consistency and high cost in keeping replicas up to date

CA Systems

  • CA Systems give up partition tolerance to ensure consistency and availability
  • What does it mean to give up partition tolerance?
    • Assuming that partitions cannot occur
    • A distributed system deployed on a single site (i.e., a datacenter) is much less likely to experience network partitions than a geographically distributed system
    • However, network partitions in a data centre are not impossible
    • What happens if a network partition takes place?
    • A design choice should be made between consistency and availability
    • Therefore, forfeiting partition tolerance is not really an option...
