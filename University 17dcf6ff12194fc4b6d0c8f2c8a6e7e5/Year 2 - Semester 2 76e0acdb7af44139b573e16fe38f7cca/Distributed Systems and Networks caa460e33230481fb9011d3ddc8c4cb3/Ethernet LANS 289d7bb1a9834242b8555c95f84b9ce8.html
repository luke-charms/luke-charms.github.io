Ethernet Networking

Ethernet in its various forms is a core enabling technology for many networks

  • Campus networks
  • Home networks

Ethernet uses a protocol called Carrier Sense Multiple Access with Collision Detection (CSMA/CD) to transmit data over a network. This protocol allows multiple devices to share the same network medium and ensures that collisions are detected and resolved.

Ethernet Frame

An ethernet frame includes:

  • 48-bit source and destination MAC addresses
  • 802.1Q tag for optional VLAN ID and frame priority

Maximum Transmission Unit is usually 1500 bytes

  • Larger messages are usually broken down which are then re-assembled by the receiver

MAC Addresses

  • Link layer addresses
    • Usually referred to as MAC addresses
    • Currently defined to be 48 bits, extensible to 64 bits
  • Needs to be unique, so:
    • 24 bits used for vendor allocations
    • Last 24 bits assigned by vendor

Building Ethernet Networks

  • Each host typically connects to a port on a switch
  • Today desktops typically 1Gbit/s Ethernet
  • Servers and backbone on 10Gbit/s Ethernet

Ethernet Switches

Receive Ethernet frames and make a decision whether to forward the frame, and if so, on which port

  • It learns Ethernet addresses of host(s) seen on each switch interface/port
  • Hosts generally only see packets sent to them, or broadcast packets

Can store and forward Ethernet frames

  • So can check for errors (CRC check) before forwarding

Some switches support more advanced functions

  • e.g., VLANs, QoS

Sending IP datagrams over Ethernet

  • Link layer receives datagrams from the network layer
  • Encapsulates these as the payload of an Ethernet frame
  • IP datagram has a source and destination IP address
  • Need to send it to the destination IP address using the Ethernet layer
  • Implies the network layer can determine the ethernet address of the destination IP address
    • However, need some method to resolve an IP address to a MAC address

This is exactly that method - used to determine the destinationâ€™s MAC address

ARP uses link layer broadcast message

  • To the special Ethernet broadcast address ff:ff:ff:ff:ff:ff

  • Seen by all hosts in the same Ethernet LAN
    • The host with the target IP address responds
  • Sender can then use result to send Ethernet frame
    • caches the result (e.g. 15 - 45 seconds on Win10)

Connecting Ethernet LANs

  • When we create an Ethernet LAN with switches, all the devices on the LAN can talk directly to each other, using their link layer (MAC addresses)
  • A LAN will also use a certain range of network layer IP addresses
    • e.g., 152.78.10.0 to 152.78.10.255 (256 addresses)
  • If we wish to connect two such LANs together, we need to use an IP (network layer) router
  • The router then forwards IP packets between the LANs
    • Routers do not forward Ethernet broadcasts; they handle IP
  • The router will typically advertise to other routers the reachability of the IP address range it directly serves

IP subnets and IP routers

An IP router sits at the edge of an Ethernet LAN

  • In a campus, typically 20-500 hosts per LAN
  • The administrators assign an IP subnet to each LAN
  • An IP address plan determines which LANs use which IP address prefixes

Intra-Building Ethernet

  • Typically uses copper twisted pair cabling

May be deployed by using:

  • Data riser
    • Vertically aligned switch/server rooms
  • Flood wiring per floor
    • To faceplates from switch room
  • Ethernet switch stacks
    • One switch port per faceplate

Inter-building is typically a fibre link

Spanning Tree (802.1d)

Bridging lots of LANs together can lead to multiple paths creating loops

IEEE 802.1d protocols can detect loops and remove paths

  • Spanning Tree Algorithm facilitates this:
    • Letâ€™s you create physical loops, without breaking network
    • Algorithm decides (elects) a root bridge/switch (lowest MAC address)
    • Determines least cost paths to the root, forming a tree of paths
    • Switches determine the root and best paths to root

VLAN (802.1Q)

Virtual switched networks over fixed infrastructure

Ethernet frame includes an optional VLAN identifier

  • 12-bit value in 802.1Q tag so 4096 different VLAN IDs
  • Can then place a switch port in specific VLAN(s)
  • Can carry multiple VLANs over one trunked uplink

Avoids need to physically re-cable and can control broadcasts to certain areas

Ethernet frame priority (802.1p)

It may be desirable to give certain traffic priority in Ethernet networks

Examples:

  • Voice over IP data
  • Network control data

Supported by using part of the 802.1Q tag

Only affects the prioritisation at the switch (however, is not end to end)
