Wireless Security

Wireless Encryption

Most contemporary networks use some sort of encryption to restrict access and ensure data integrity

Open hotspots are used to provide complimentary / public access

Man in the middle attacks!

WEP

Wired Equivalent Privacy - the original method of encryption for 802.11 Wireless Networks

Intended to provide data confidentiality comparable to a wired network

40-bit or 104-bit key and 23-bit IV

Uses the RC4 stream cypher

Proved to be insecure in the early 2000s

  • In 2005, the FBI demonstrated that a busy WEP protected network could be cracked in 3 minutes!

WPA/WPA2/WPA3

Wifi Protected Access

  • An intermediate solution to the weakness of WEP
  • Uses Temporal Key Integrity Protocol (TKIP)

Wi-fi Protected Access II

  • More secure and more complex than WPA
  • Uses AES as the encryption algorithm

Wi-fi Protected Access III

  • Same encryption “strength” as WPA2
  • Improves security of initial key exchange

WPA-Personal: uses pre-shared keys

WPA PSK Vulnerabilities

Attacks mainly focused, until recently, on the human element or other weak/predictable systems

  • e.g., weak user-supplied passwords vulnerable to a dictionary attack

Weak auto-generation of ISP-set default keys

WPS

Wifi Protected Setup

Intended to make it easier to connect to a WPA-protected network

User enters an 8-digit pin or presses a button on the access point for initial connection

BUT, pin can be brute-forced quickly 😢

Prevailing recommendation is to disable WPS!

KRACK - October 2017

Key Reinstallation AttaCK

Replay attack that resets a number used during authentication

Allows an attacker to (gradually) work out the full keychain and decrypt traffic

Weakness is in the standard itself, not implementation specific

Linux (WPA-Supplicant) is more vulnerable as it can set an all-zero encryption key

Requires both clients and access points to be updated

  • Not likely to happen for home access points!

KrØØk

Vulnerability in some WiFi chipsets (Broadcaom, Cypress, Qualcomm,…) found in devices like raspberry pi, iPhone 8, etc…
